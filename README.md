# grok-ozempic

**SNN-inspired quantization for Grok-scale Mixture-of-Experts models** — thin down the weights without compromising routing accuracy.

This crate turns heavyweight Grok checkpoints into a spiking-friendly representation using a ternary SNN encoding and FP16 passthrough where necessary.

**Think of it as Ozempic for Grok — less bulk, same routing story.**

---

## Key features

### Authoritative classification via `xai-dissect` manifests

Instead of brittle substring heuristics, `grok-ozempic` reads a JSON manifest generated by [`xai-dissect`](https://github.com/rmems/xai-dissect).

Tensors are assigned to one of three precision tiers:

- `preserve`
- `fp16`
- `ternary_snn`

Classification is driven by **segment-anchored glob rules**, not sloppy substring matching.

The loader validates:

- schema version
- tensor naming convention
- precision tier strings

and will **hard-fail** on invalid manifest data.

Manifest delivery precedence is:

1. explicit config path
2. `GROK_OZEMPIC_MANIFEST`
3. embedded Grok-1 baseline
4. legacy substring heuristic fallback

---

### Streaming three-pass quantization pipeline

Quantization is **out-of-core**.

Only one tensor is ever held in memory at a time.

Pipeline:

1. **Manifest pass**  
   Scan shards and record metadata only

2. **Header pass**  
   Write the GOZ1 header and tensor table

3. **Data pass**  
   Map each tensor, quantize or pass through, and stream bytes into the output file

This keeps memory usage low even for very large checkpoints.

---

### Ternary SNN encoding with per-tensor thresholds

For `ternary_snn` candidates, the quantizer applies a saliency threshold:

```text
τ = gif_threshold × rms(layer)
```
